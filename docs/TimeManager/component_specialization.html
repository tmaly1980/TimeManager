Time Manager: Task Specialization

The Task Specialization module takes a group's task categories and lets certain people within that group be designated to handle those types of tasks. In other words, it simplifies figuring out who should do a task, purely by know what kind of task it is.

Goals:

The Task Specialization module helps you:

  • Figure out who should do a task based upon what it involves
Overview:
Task Category Specification:
Task categories are defined for each group of users. The task categories may have stages, further describing the exact progress level.
User/Category Association:
You can define which task categories a user can and cannot do via the user edit page. By default, users can perform newly added task categories.
Task Assignment:
Once a task gets assigned a category, the list of possible owners will automatically change to who is qualified/designated for that task category.
Features:

The Task Specialization module consists of the following major parts (click on the name for further details):

  • Task Category Specification: Designating what kinds of things people in a group do
  • Task Category Association: Designating which task categories a person is and is not qualified for
Getting There:
Task Category Specification:
To specify which task categories that belong to a group/team, you must be the group's manager or site administrator. Go to the Administration Console, and click on the group. From there, task categories and their respective stages can be added, changed, or removed.
Task Category Association:
A site administrator designates who can perform what task categories, by going to the user's profile page from the Administration Console.
