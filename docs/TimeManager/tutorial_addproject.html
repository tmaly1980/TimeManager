Product Tutorial: Adding a Project

Projects can be created by anyone, and can include anyone else as a participant, independent of normal managerial rights. Adding a project takes effect once you click on the 'Add' button. The project overview screen should then show up in the main window.

Sections:

  1. Opening the 'Add Project' Window
  2. Title
  3. Product
  4. Project Manager
  5. Objective
  6. Start / Due Dates
  7. Priority
  8. Budget
  9. Participants
  10. Resources
  11. Contingency Plan
  1. Opening the 'Add Project' Window:

    2. From the Project Management Console, you can click on 'Add Project' to open a window to add the project.

  2. Title:

    3. The title gives a short description as to what the project involves, typically a product name and version.

  3. Product:

    4. Projects typically involve performing revisions on products -- whether something sold to customers, or an internal "unit of operation" (such as a component of the IT infrastructure). If there is a specific version, that may be selected as well.

  4. Project Manager:

    5. The project manager is the one responsible for a project to finish according to specifications. They can add milestones, tasks, and participants.

  5. Objective:

    6. The 'objective' defines the overall set of goals one wants to accomplish by this project. It should be in terms of how to do things, rather than the technical specifications of what to do (which is left up to the tasks).

  6. Start / Due Dates:

    7. The start and due dates specify when you want the project to start and end, respectively. Milestones and tasks may extend before or after these dates, in which the project manager needs to do further investigation/work.

  7. Priority:

    8. The priority specifies how important the project is, relative to other projects.

  8. Budget:

    9. The budget specifies how much money is available to accomplish the project. The expenses that deplete the budget are calculated from the cost of each task. A blank budget means unlimited funds, whereas a budget of $0 means no funds at all.

  9. Participants:

    10. The participants list which users can be assigned tasks by the project manager. The original list contains all the users in the system, not just users you have managerial rights over. Participants are marked in the right-hand list, and added by clicking on a name on the left-hand list and then clicking the right arrow. They are removed by clicking on a member in the right-hand list and clicking the left arrow.

  10. Resources:

    11. The resources are external items needed to complete the project. Typing in some text in the text field and clicking on 'Add' or 'Replace' will put it in the list. Highlighting an entry and clicking on 'Remove' will remove it from the list.

  11. Contingency Plan:

    12. The contingency plan documents an emergency recovery plan in case the entire project must be aborted.
