Product Tutorial: Searching for Projects

Searching for projects provides an easy way to find out what work your employees have, as well as what you have coming up. Only criteria explicitly entered (i.e. non-blank entries) will be taken into consideration.

Sections:

  1. Opening the Search page
  2. Project Name
  3. Product
  4. Status
  5. Project Manager / Participant
  6. Dates
  7. Group By
  8. Show Milestones
  1. Opening the Search page:

    2. By clicking on the 'Search' link under the Project Management Console ('Projects' tab), one gets the search form for projects.

  2. Project Name:

    3. One can specify the project by name from the drop-down list, if it is known. Only the projects of people you can manage (including yourself) will show up.

  3. Product:

    4. One can specify the product (or 'None' if there is none), with or without a specific version.

  4. Status:

    5. When searching by the status, it is automatically calculated from that of the contained tasks.

  5. Project Manager / Participant:

    6. One can search by the project manager or a participant within the project.

  6. Dates:

    7. One can search on using one or two dates, as to when a project was started, worked on or due. The dates can specify the exact day or the week of what was given.

  7. Group By:

    8. One can aggregate the results into sections, as specified by the 'Group By' field. Any of the search criteria can be used, as well as the start or due date's week.

  8. Show Milestones:

    9. The 'Show Milestones' specifies whether milestones from within the project should be displayed or not.
