Product Tutorial: Managing task dependencies/subtasks

Subtasks designate parts of a task divided and distributed to someone else. Task dependencies are separate tasks that need to be finished prior to a task being started or completed. Providing a mechanism to show and alter this structure helps everyone understand just what really is involved with finishing a task. All changes will not take effect until you click on 'Save' on the parent task.

Sections:

  1. Opening the task
  2. Adding new subtasks / dependencies
  3. Adding existing subtasks / dependencies
  4. Unlinking a subtask / dependency
  5. Browse All
  6. Gantt Chart
  7. Marking dependencies
  1. Opening the task:

    2. From the task browse or task search pages, one must first open a task.

  2. Adding new subtasks / dependencies:

    3. By clicking on the 'New' button next to the subtasks/dependencies section, one can create a subtask or dependency. Once the task is created, it will be added to the current list. If you want something to be considered a dependency, you should highlight it and click on 'Must complete before parent'. No changes will take effect until you click on the 'Save' button (at the top or bottom of the screen).

  3. Adding existing subtasks / dependencies:

    4. By clicking on 'Existing', you get presented a search page, where you then can click on 'Link' next to a task to add to your list. After closing the new window, you can designate a dependency via 'Must complete before parent'. After clicking on the 'Save' button, your changes will take effect.

  4. Unlinking a subtask / dependency:

    5. By highlighting a subtask / dependency and then clicking on 'Unlink', you remove it from the list. It is NOT removed from the system, it just becomes independent of the current task. Changes take effect once you click on 'Save'.

  5. Browse All:

    6. Clicking on 'Browse All' presents a search result page, with the option to perform bulk changes on those tasks.

  6. Gantt Chart:

    7. Clicking on 'Gantt' presents a Gantt timeline chart of the task and it's subtasks / dependencies. It provides a better graphical understanding of how the progress is and how the tasks depend on each other.

  7. Marking dependencies:

    8. After linking an external (existing) task to yours, you can click on 'Must complete before parent' to set it as a dependency task.
