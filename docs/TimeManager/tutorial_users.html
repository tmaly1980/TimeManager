Product Tutorial: Managing User Accounts

User profiles (accounts) are managed via the Administration Console. You can also access your profile from the main menu at any time. From the Administration Console, you can manage only those you were granted to via groups or explicit memberships (via the site administrator). User accounts are used by individuals, containing information such as their full name, job title, email, hours per week available, site admin access and explicit managers. No changes take effect until 'Add' or 'Save' is clicked on.

  1. Accessing your profile:

    2. You can access your profile at any time by clicking on your name in the main menu.

  2. Accessing your team's profiles:

    3. By going to the Administration Console (via the 'Admin' menu tab), you can get a list of which users you are allowed to manage and see. Simply click on a user's name in order to get to their profile.

  3. Email:

    4. The email specified is used when major notification occur, such as when a task closes or gets assigned.

  4. Hours/Week Available:

    5. The hours per week available specify the threshold as used with the workload comparison module to determine at what point the user is 'overworked'. It defaults to 40 hours per week if not set to some other value.

  5. Assignment By / Explicit Managers:

    6. The 'explicit managers' section defines a list (on the right-hand side) as to who can assign tasks to this particular user without being their group manager. They are added via highlighting the member in the left-hand list and clicking on the right arrow, and removed by highlighting a member in the right-hand list and clicking on the left arrow.

  6. Site Administrator:

    7. The 'Admin' checkbox specifies whether the user is a site administrator or not. This provides complete access to add, edit and remove tasks, projects and milestones, with no regard to an organizational chart. This access should only be given to a few people if any, such as IT help desk personnel.

  7. Password:

    8. The password for this user to log in is specified twice upon creation. If the password needs to be changed, type it in twice. Leaving the fields blank means that the password keeps the value it has now, and will not get erased.

  8. Workload Comparison:

    9. By selecting users in the checkboxes and then clicking on 'View/Compare Workload', or by clicking on 'Workload' in a user's edit screen, one can get a list of what their workloads look like versus each other, and over time.

    See Also:
    • Workload Comparison
  9. Task Categories:

    10. The right-hand list displays what task categories (from the person's groups) that they can/should handle. Clicking on something on the right-hand list and then the left arrow will remove it from their 'abilities' list. Moving something from the left list to the right list adds it back.

  10. Deleting Users:

    11. You can delete a user by clicking on 'Delete' from within their edit page. If they have any tasks assigned to them that you wish to reassign, it is RECOMMENDED to do that first, as after the account is deleted, it is IMPOSSIBLE to search for their tasks!
